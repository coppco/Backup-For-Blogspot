由于hexo使用的是markdown格式的文件,而我又不会使用markdown,所以第一篇微博就来学习一下markdown的使用和hexo的一些命令吧!

Markdown的基本语法

1、分段: 两个回车

2、换行: 两个空格 + 回车

3、标题: 可以使用#或者文字下一行使用两个=和两个-

  • 方式1,使用# ~ ######文字, 来显示标题, #的个数表示几级标题,需要注意的是:#和文字之间需要有空格,而且在段落中使用是无效的。

    四个

  • 方式2, 使用=和-

    使用=

    使用-

    4、粗体和斜体: _*一个包裹的是斜体, 两个包裹的是粗体。

斜体 粗体 斜体 粗体

5、引用: 通过在文字开头添加 > 来表示。 (当 > 和文字之间添加五个空格blank时,样式会变化,变成黑底)

引用无空格 

引用五个格

6、代码区域: 四个空格开头或者使用`把代码包住 (这个是~下面的符号,不是单引号)

return YES;

7、列表: * + -是无序列表,1. 是有序列表, 注意后面都是有空格。

  • 无序*
  • 无序+
  • 无序-
  1. 有序数字.

8、链接Links: Markdown中有两种方式，实现链接，分别为内联方式和引用方式。

  • 内联方式 不会的问题先问问[百度], 再问别人!(http://www.baidu.com/)。
  • 引用方式 欢迎访问我的github和简书。

9、图片image: 图片和链接类似,有内联方式和引用方式, 地址可以是本地文件也可以是网络URL。

  • 内联方式使用,本地图片
  • 引用方式,使用网络图片

10、脚注: 实现方式如hello[^hello]

11、下划线: 在空白行下方添加三条-横线

hexo的一些命令

1、hexo new “文章名称”: 新建一个md文件,在 hexo/source/_posts/目录下

2、hexo clean: 当source文件夹中文件改变的时候,这个操作可以清除生成的public文件夹

3、hexo g: 重新生成静态文件夹public

4、hexo s: 开启本地服务打开ttp://0.0.0.0:4000/预览

____5、hexo d: 部署到远程git上面, 主要是把生成的public文件夹上传到git

后记

使用hexo+github搭建个人博客的时候,难免遇到使用不同的电脑写博客,我是这样解决的:把博客所在的目录也使用一个git仓库存储,这样一些资源、主题的配置可以得到备份. 以后使用不同的电脑就clone下来,当然环境node、hexo、git等都是需要安装的,然后进入博客文件夹使用 npm install 之后npm install hexo-deployer-git –save来配置远程git仓库,之后就可以飞了!!!!
第一次使用markdown写这个,还是比较生疏!
